1M5 OS

The mission's primary means to ensure censorship-resistant private communications is through its own operating system (OS) based on Linux that routes around blocks using unblocked networks and peers as relays. The following are some features of this operating system.

Censorship Resistant Routing

Uses TOR, I2P, and 1DN (Bluetooth, WiFi-Direct, Satellites, Full-Spectrum Radio, LiFi) for privacy routing around censorship attempts by using unblocked networks and peers as relays.

Micro-Services DApp Platform

1M5 is a micro-service platform for developing decentralized applications within a Service-Oriented Architecture (SOA) and a Staged Event-Driven Architecture (SEDA) supporting routing between services using a routing service that implements Enterprise Integration Patterns (EIP) and Decentralized Routing Algorithms. Services can be managed independently of the application.

I2P

The Invisible Internet Project - an anonymous internet overlay network of about 65k peers. Provides garlic-routing, layered encryption, and uni-directional channels for IP anonymity. Best for communicating P2P with other I2P users.

TOR

The Onion Router (TOR) - an anonymous internet overlay network of about 7k nodes. Provides onion-routing, layered encryption, and bi-directional channels for IP anonymity. Best for accessing clearnet web sites/services.

1DN

A direct wireless ad-hoc network using Bluetooth, WiFi-Direct, Satellites, Full-Spectrum Radio, and LiFi to support private communications outside of the internet.

Decentralized IDentifier

Pluggable, self-sovereign, decentralized, people and/or automation-controlled universal ID, called DID or Decentralized IDentifier using OpenPGP. Only you have the keys. Build reputation with them.

Key Ring

Encryption and Signing keys kept safe on specialized flash drives where once added they can not be read nor changed. Encryption and signing happens on-drive only. OpenPGP used for standards support.

Info Vault

The OS allows for the user to grant access control to their personal information on their device if desired. Access can be revoked when its no longer desired for them to have it. This will end widespread data breaches. The OS runs purely in memory not accessing the hard-drive, only the physical external drives you provide.
